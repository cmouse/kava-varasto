# Security Policy

## Supported versions

Only the latest release is supported. This is a small, single-deployment
club application, not a library with maintained release branches — apply
security fixes by upgrading to the latest tag.

## Reporting a vulnerability

Please report security issues privately using GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
(the "Security" tab on this repository → "Report a vulnerability").

Do not open a public issue for security reports.

We'll acknowledge reports as soon as possible and follow up once a fix is
available.
